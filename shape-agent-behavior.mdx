---
title: "Shape agent behavior"
description: "Refine an agent after the first draft by updating instructions, triggers, and tools."
---

Your first draft is the starting point. Use the agent side panel and chat to make it more precise.

<Note>
  Depending on the type of agent you built, the side panel may be labeled **Workflow** or **Knowledge**.
</Note>

## Update instructions

Instructions are the agent's plan of action.

- Keep them specific.
- Tell the agent what inputs to use.
- Tell the agent what output to produce.
- Remove vague language when the agent behaves inconsistently.

If the current behavior is wrong, describe the change in chat and then review the updated instructions.

## Add triggers

Triggers let an agent run automatically when an event happens.

In the **Workflow** panel, look for the **Triggers** section to review the triggers already attached to the agent.

If no trigger exists yet, you can ask the agent in chat what triggers are available and then configure the one you want.

## Review tools and subagents

Open **Connected Apps** to see which apps the agent can use.

If the agent uses subagents, they appear in the same side panel so you can inspect them separately.

## Keep the scope tight

- Give each agent one clear responsibility.
- Split large workflows into smaller agents if the behavior becomes hard to control.
- Re-test the agent after each meaningful change.
